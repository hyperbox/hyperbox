Getting started

IMPORTANT NOTICE

The software is still in testing phase and should not be used for production environments.
It is assumed that you have a strong knowledge of Virtualbox and system administration.
Such topics will not be supported, to allow focus on the Hyperbox software itself.

Introduction

Hyperbox is designed to be the manager of your virtualized infrastructure, and is best used on a dedicated host where no other tools accessing and/or controlling the hypervisor are running.
It is composed of 2 parts:

  • The Server where all the work take places and which interacts with the hypervisor and abstract out all the details to present a common layer that can be used to manage hybrid environments, e.g. Virtualbox and VMware ESXi.
  • The Client which send the user input to the server for processing and allow to interact with the Server and the Hypervisor

Server

Introduction

The server aims to provide centralized user management with authentication, authorization and auditing. This means that a dedicated system user should be used to run the hypervisor - opposite of what Virtualbox does by default per example.
Keeping this in mind, the following instructions are given for Debian-based systems, prefered OS for Hyperbox.
If you are using another Linux distro, manual installation can be performed using the zip package, coping the server files to a location of your choice, and installing the init.d script. Note that the init.d script will need minor adjustements.
If you are using Windows, the installer will copy the server files and install the service. Starting from there, the instructions are the same for configuration and logs review.

Client

Installation

To install the client, simply download its installer and run it. For Debian & Fedora based distributions, run the following : sudo sh ./hbox_client-*.run For other distros, use the zip package and perform a manual install.

For Windows, run the installer.

The Client can then be launched via the Start Menu/Main menu or by creating a shortcut poiting to <INSTALL_DIR>/hyperbox(.exe) and setting the working dir to <INSTALL_DIR>
