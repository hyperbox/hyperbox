Advanced Configuration

Common

Name Command-Line option Environment Variable Daemon Config File Purpose
Log File -Dlog.file HBOX_LOG_FILE log.file Define where Hyperbox will log messages. Relative paths will start from the installation directory.
Log Output Level -Dlog.level HBOX_LOG_LEVEL log.level Define the maximum loging level for Hyperbox. A level include all the levels above (in terms of verbosity). The following levels are available in increasing verbosity order :
FatalException - Only log exceptions stack traces that caused the server to stop. This is currently not used.
Exception - Log exceptions stack traces that are not fatal. This is tightly linked to Error level as each error will also produce an exception entry.
Error - Log errors that occured. This is tightly linked to Exception level as the message will be the exception message.
Warning - Log warnings.
Info - Log regular info messages. This is the default level.
Verbose - Log precises info messages.
Debug - Log debug info for each server activity, task and client request: This can be quite verbose and should not be kept on in live environement due to the performance penality.
Tracking - Log code tracking info: This is very verbose and WILL inflict a high performance penality but will provide very detailed info for debugging.

Server

Name Command-Line option Environment Variable Daemon Config File Purpose
Server Name -Dserver.name HBOX_SERVER_NAME server.name Configure a name for the server that will be display in the client.
The hostname will be used as default value.
Kryonet Connector Port -Dfront.kryonet.port HBOX_FRONT_KRYONET_PORT front.kryonet.port Define the listening port for the default client connector using Kryonet.
Default value is 45612
